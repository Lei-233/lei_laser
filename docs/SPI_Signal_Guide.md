# DAC8562 SPI信号测试指南

## 概述

`spi_test` 是专为逻辑分析仪验证设计的DAC8562 SPI信号测试工具。通过生成多种可预测的波形和数据模式，方便验证SPI通信的正确性和硬件连接质量。

## 编译

```bash
cd /home/lei/lei_laser/user
make spi_test
```

## 部署到开发板

```bash
scp spi_test root@<开发板IP>:/tmp/
```

## 使用方法

### 基本语法

```bash
./spi_test <模式> [频率Hz] [样本数]
```

### 参数说明

| 参数 | 说明 | 默认值 | 范围 |
|------|------|--------|------|
| 模式 | 测试模式编号 (0-5) | 必填 | 0-5 |
| 频率Hz | 信号更新频率 | 1000 | 1-10000 |
| 样本数 | 发送样本总数，0=无限 | 0 | ≥0 |

### 测试模式详解

| 模式 | 名称 | 描述 | 适用场景 |
|------|------|------|----------|
| 0 | 方波模式 | 交替发送 0x0000 和 0xFFFF | **推荐首选** - 最明显的信号，易于观察时序 |
| 1 | 三角波模式 | 0x0000 → 0xFFFF 线性递增/递减 | 验证数据连续性和DAC线性度 |
| 2 | 数据模式 | 循环发送特定测试图案<br>(0xAAAA, 0x5555, 0xFFFF, 0x0000等) | **推荐** - 验证数据完整性和位传输正确性 |
| 3 | 交替通道模式 | 单独激活 DAC-A 或 DAC-B | 测试双通道独立性 |
| 4 | 计数器模式 | 递增计数序列 | 验证长时间数据完整性 |
| 5 | 单步模式 | 发送单个中点值 (0x8000) 后停止 | **推荐** - 精确捕获单次SPI传输 |

## 使用示例

### 1. 逻辑分析仪快速验证（推荐）

**场景：首次验证SPI信号是否正常**

```bash
# 单步模式 - 发送一次后停止，方便捕获
./spi_test 5
```

**预期结果：**
- 逻辑分析仪应捕获到两组24位SPI传输
- 传输1: `0x18 0x80 0x00` (写DAC-A, 值=0x8000)
- 传输2: `0x19 0x80 0x00` (写DAC-B, 值=0x8000)

### 2. 验证SPI时序（推荐）

**场景：检查SCLK、MOSI、CS信号时序关系**

```bash
# 方波模式, 10Hz 慢速更新, 便于观察
./spi_test 0 10
```

**检查要点：**
- CS片选在传输前拉低，传输后拉高
- SCLK时钟频率 ≤ 20MHz（设备树配置）
- MOSI数据在SCLK上升沿有效（SPI Mode 1）
- 每组传输24个SCLK周期

### 3. 验证数据正确性（推荐）

**场景：确认发送的数据在总线上无误**

```bash
# 数据模式, 100Hz, 发送100个样本
./spi_test 2 100 100
```

**验证方法：**
1. 用逻辑分析仪的SPI解码功能解析MOSI数据
2. 检查是否依次接收到：
   - `0x18 0xAA 0xAA` / `0x19 0x55 0x55`
   - `0x18 0x55 0x55` / `0x19 0xAA 0xAA`
   - `0x18 0xFF 0xFF` / `0x19 0x00 0x00`
   - ...（循环）

### 4. 长时间稳定性测试

**场景：验证信号质量和硬件连接可靠性**

```bash
# 方波模式, 1KHz, 无限循环 (Ctrl+C 停止)
./spi_test 0 1000
```

**观察要点：**
- 信号边沿是否干净，无过冲/振铃
- 数据是否有错位或丢失
- 长时间运行是否稳定

### 5. 高速性能测试

**场景：测试最高更新速率**

```bash
# 方波模式, 10KHz (100us更新间隔)
./spi_test 0 10000
```

**注意：** DAC8562支持最高50MHz SPI时钟，但实际更新速率受CPU和驱动影响。

## 逻辑分析仪连接

### 最小连接（必需）

| 信号 | 功能 | 连接位置 |
|------|------|----------|
| SCLK | SPI时钟 | SPI3_SCLK |
| MOSI | 主机输出数据 | SPI3_MOSI |
| CS   | 片选 | SPI3_CS0 |
| GND  | 地 | 板上任意GND |

### 完整连接（推荐）

额外增加：
- **LDAC** (可选，DAC8562立即更新模式下不使用)
- **VOUTA** 和 **VOUTB** (DAC模拟输出，需模拟通道验证)

### 采样率建议

- **SPI时钟 20MHz** → 逻辑分析仪采样率 ≥ **100 MSPS**
- **推荐**: 200 MSPS 或更高（提供更好的波形细节）

## 常见问题排查

### 1. 无法打开 /dev/galvo

**错误信息：**
```
错误: 无法打开 /dev/galvo: No such file or directory
```

**解决方法：**
```bash
# 检查模块是否加载
lsmod | grep galvo

# 如未加载，手动加载
insmod /path/to/galvo.ko

# 检查设备节点
ls -l /dev/galvo
```

### 2. galvo_set_ab 失败

**可能原因：**
- SPI总线被占用
- 驱动初始化失败
- 硬件连接问题

**检查步骤：**
```bash
# 查看内核日志
dmesg | tail -20

# 检查SPI设备
cat /sys/bus/spi/devices/spi*/modalias
```

### 3. 逻辑分析仪无信号

**检查清单：**
- [ ] 探头是否正确连接到SCLK/MOSI/CS
- [ ] GND是否共地
- [ ] 逻辑分析仪触发电平是否正确（3.3V CMOS）
- [ ] 程序是否正在运行（检查终端输出）

### 4. SPI解码数据异常

**可能原因：**
- SPI模式配置错误 → 应为 Mode 1 (CPOL=0, CPHA=1)
- 时钟相位不匹配 → 检查CPHA设置
- 采样率过低 → 提高到200 MSPS以上

## DAC8562 SPI协议快速参考

### 帧格式（24位）

```
[CMD 8位][DATA_HIGH 8位][DATA_LOW 8位]
```

### 常用命令

| 命令 | 功能 | 数据格式 |
|------|------|----------|
| 0x18 | 写DAC-A并更新 | 16位DAC值 (0x0000-0xFFFF) |
| 0x19 | 写DAC-B并更新 | 16位DAC值 |
| 0x1F | 同时写AB并更新 | 16位DAC值 |
| 0x28 | 软件复位 | 0x0001 |
| 0x38 | 内部参考使能 | 0x0001 (增益=2) |

### 电气特性

| 参数 | 值 |
|------|------|
| SPI时钟最大 | 50 MHz |
| SPI模式 | Mode 1 (CPOL=0, CPHA=1) |
| 逻辑电平 | 2.7V - 5.5V CMOS |
| 输出范围 | 0 - 5V (内部参考增益=2) |
| 分辨率 | 16位 (65536级) |
