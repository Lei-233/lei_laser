# 工业级高性能激光打标系统驱动 (lei_laser)

<div align="center">
以底层性能为核心，基于 RK3566 开发板的纯异步高并发嵌入式 Linux 激光数控系统。
</div>

---

## 📖 项目简介

本项目旨在打破传统的“主控PC + 辅助单片机(MCU)”控制架构，在一块跑着标准 Linux 的主板上实现高精度硬实时的机械数控。通过对内核框架的深度魔改，去除了普通外设接口由于分时系统进程调度而导致的延迟和抖动，实现了振镜的偏转运动与激光发射信号的**微秒级精确同步控制**。这不仅是一整套稳定可用的打标驱动端，也是深入理解 Linux 底层并发隔离与高精度时钟的极佳案例。

## ✨ 核心技术亮点

本项目最核心的技术突破在于“应用层算力与内核层硬实时脱耦”：
* **hrtimer 纳秒级高精度定时器**：抛弃了传统的 Jiffies 节拍机制，直接绑定底层硬件高速计数器，在非实时 Linux 环境中强制开辟“微秒级硬实时快车道”。
* **全面异步（spi_async）的非阻塞架构**：在内核中断上下文中严格遵循"绝不休眠"的铁律。弃用了易触发调度的 `spi_sync` 和 `mutex`，改用全异步接口与原子错误追踪。
* **SPI 双缓冲乒乓切换队列**：消除了 DMA 数据搬运时与高频中断定时器覆写产生的内存竞态冲突风险。
* **vmalloc 大缓冲突破碎片化**：摒弃物理连续内存依赖，系统将百万级图形数据包裹瞬间下沉内化，即使在主板长时间高负载产生大量内存碎片下仍不崩溃。
* **定制化驱动通信基石**：抛弃 `spidev` 通用挂载，亲自在内核最底端统合 SPI 控制与 PWM 控制，并通过专属 `ioctl` （5 大指令群）建立用户与内核态的安检通讯通道。

## 🛠️ 项目构建与部署

### 1. 软件依赖与交叉编译环境
- **硬件平台**：RK3566 (泰山派开发板)
- **操作系统**：Buildroot 定制 Linux (或任何兼容 4.19 / 5.10 的内核构建)
- **交叉编译链**：`aarch64-linux-gnu-gcc` (推荐 gcc-linaro-6.3.1)

### 2. 构建工程

克隆本项目到编译主机的某个目录，分别编译内核模块和用户程序：

```bash
# ====== 第一步：编译内核模块 (galvo.ko) ======
cd kernel
make clean
make
# 产物：galvo.ko

# ====== 第二步：编译用户交互程序 ======
cd ../user
mkdir -p build && cd build
cmake ..
make
# 产物：galvo_app, uart_controller
```

### 3. 上板端部署与启动

将产生的文件（`galvo.ko`, `galvo_app`, `uart_controller`）通过 scp 发送到开发板，然后在板卡上执行：

```bash
# 挂载设备树内核专属字符设备模块
insmod galvo.ko 

# 模式A：独立打图测试 (0为画方形，1为画圆)
./galvo_app 1 

# 模式B：接入业务链路测试，作为完整串口伺服后端
./uart_controller /dev/ttyS3
```

> **提示：** `uart_controller` 已完全支持 **Grbl 1.1f** 接口标准规范，主机可以通过 LaserGRBL / LightBurn 等软件通过波特率 `115200` 直接联调输出图像至本机。
